## 📝 示例 1：日常笔记检查（3Q-Lite）

### 场景
用户写完每日学习笔记，想快速检查质量。

### 对话流程

```
用户：请帮我检查这个笔记的 3Q

[提交文档：2026-03-28 学习总结.md]
内容：今天学习了 3Q 融合方案...（约 800 字）

系统：📊 文档分析结果
      - 文档类型：学习笔记
      - 字数：800 字
      - 重要性评分：25/100（低）
      
      ✅ 推荐使用 3Q-Lite（三问框架）
      ⏱️ 预计时间：5-10 分钟
      
      是否开始检查？

用户：开始

系统：🔍 开始 3Q-Lite 检查...

      ### 1. 逻辑 Q（__/5 分）
      问题：论据是否支持论点？
      检查：...
      
      ### 2. 用户 Q（__/5 分）
      问题：解决了什么痛点？
      检查：...
      
      ### 3. 竞争 Q（__/5 分）
      问题：与现有方案比有什么优势？
      检查：...
      
      📊 总分：13/15（B 级）
      
      💡 改进建议：
      1. ...
      2. ...

用户：好的，谢谢！
```

## 📝 示例 3：里程碑验收（3Q-Max）

### 场景

### 对话流程

```
用户：项目要发布了，帮忙做 3Q 验收

[提交文档：DivePast v2.0 发布计划.md]

      ⏱️ 预计时间：2-4 小时
      
      是否开始检查？

用户：开始

系统：🔍 开始 3Q-Max 检查...

      ## 阶段 1：事前 3Q（15 分钟）
      
      ### 任务理解确认
      - [✓] 任务目标是否清晰？
      - [✓] 交付标准是否明确？
      - [✓] 时间要求是否合理？
      
      ### 质量标准定义
      - [✓] 质量指标是否定义？
      - [✓] 验收标准是否清晰？
      - [✓] 风险阈值是否设定？
      
      ### 风险识别
      - [✓] 主要风险有哪些？
      - [✓] 应对措施是否准备？
      - [✓] 应急预案是否制定？
      
      ✅ 阶段 1 完成
      
      ## 阶段 2：事中 13 检查点（1-3 小时）
      ...
      
      ## 阶段 3：事后 3Q（15 分钟）
      ...
      
      📊 验收结论：通过
      ✅ 所有检查点合格
      
      🎉 恭喜发布！
```
