## 📚 框架列表

### 3Q-Lite（三问框架）

**定位**: 日常文档快速检查

**框架内容**：
```markdown
## 3Q-Lite 检查清单

### 1. 逻辑 Q
- [ ] 论据是否支持论点？
- [ ] 有无逻辑漏洞？
- [ ] 结论是否合理？

### 2. 用户 Q
- [ ] 解决了什么痛点？
- [ ] 有无可感知的价值？
- [ ] 表达是否清晰？

### 3. 竞争 Q
- [ ] 与现有方案比有什么优势？
- [ ] 独特价值是什么？

### 评分
- 逻辑 Q: __/5
- 用户 Q: __/5
- 竞争 Q: __/5
- **总分**: __/15
```

**适用场景**：
- 日常笔记
- 会议纪要
- 学习计划
- 日志

**预计时间**：5-10 分钟

### 3Q-Pro（六问框架）

**定位**: 重要文档深度检查

**框架内容**：
```markdown
## 3Q-Pro 检查清单

### 元认知层（新增）

#### 1. 元认知 Q
- [ ] 我对这个任务的理解有 95% 把握吗？
- [ ] 哪些地方还需要澄清？
- [ ] 我是否在逃避提问？

#### 2. 标杆 Q
- [ ] 这个领域前 1% 的人会怎么做？
- [ ] 他们的思维模式和我有什么不同？
- [ ] 我可以在哪些方面向他们学习？

#### 3. 挑战 Q
- [ ] 如果我的假设完全错误，会怎样？
- [ ] 最强反对意见是什么？
- [ ] 有哪些潜在风险未考虑？

### 基础层（继承）

#### 4. 逻辑 Q
- [ ] 论据是否支持论点？
- [ ] 有无逻辑漏洞？
- [ ] 结论是否合理？

#### 5. 用户 Q
- [ ] 解决了什么痛点？
- [ ] 有无可感知的价值？
- [ ] 表达是否清晰？

#### 6. 竞争 Q
- [ ] 与现有方案比有什么优势？
- [ ] 独特价值是什么？

### 评分
- 元认知 Q: __/2.5
- 标杆 Q: __/2.5
- 挑战 Q: __/2.5
- 逻辑 Q: __/2.5
- 用户 Q: __/2.5
- 竞争 Q: __/2.5
- **总分**: __/15
```

**适用场景**：
- PRD/方案
- 技能文档
- 学习笔记（深度）
- 项目总结

**预计时间**：30-45 分钟

### 3Q-Max（三阶段框架）

**定位**: 项目里程碑验收

**框架内容**：
```markdown
## 3Q-Max 检查清单

### 阶段 1：事前 3Q（15 分钟）

#### 任务理解确认
- [ ] 任务目标是否清晰？
- [ ] 交付标准是否明确？
- [ ] 时间要求是否合理？

#### 质量标准定义
- [ ] 质量指标是否定义？
- [ ] 验收标准是否清晰？
- [ ] 风险阈值是否设定？

#### 风险识别
- [ ] 主要风险有哪些？
- [ ] 应对措施是否准备？
- [ ] 应急预案是否制定？

### 阶段 2：事中 13 检查点（1-3 小时）

#### 需求完整性
- [ ] 需求是否完整？
- [ ] 优先级是否清晰？
- [ ] 验收标准是否可衡量？

#### 技术方案
- [ ] 技术方案是否合理？
- [ ] 技术风险是否评估？
- [ ] 备选方案是否准备？

#### 测试覆盖
- [ ] 测试用例是否完整？
- [ ] 边界情况是否覆盖？
- [ ] 性能测试是否执行？

#### 文档完整性
- [ ] 文档是否完整？
- [ ] 注释是否清晰？
- [ ] 更新日志是否记录？

...（共 13 个检查点）

### 阶段 3：事后 3Q（15 分钟）

#### 交付物验收
- [ ] 交付物是否符合标准？
- [ ] 质量问题是否修复？
- [ ] 用户验收是否通过？

#### 经验总结
- [ ] 做得好的地方？
- [ ] 需要改进的地方？
- [ ] 经验教训是否记录？

#### 改进建议
- [ ] 流程改进建议？
- [ ] 工具改进建议？
- [ ] 技能提升建议？

### 验收结论
- [ ] 通过
- [ ] 有条件通过（需修复 minor 问题）
- [ ] 不通过（需重新检查）
```

**适用场景**：
- 项目里程碑
- 产品发布
- 技能交付
- 重要验收

**预计时间**：2-4 小时
