# 3Q-Unified v1.0 - 统一 3Q 质量检查技能

## 📖 Overview（概述）

**3Q-Unified** 是 DivePast 质量保障体系的统一入口，整合了所有 3Q 相关技能。

通过**智能识别引擎**自动分析文档类型和重要性，匹配最合适的检查框架（3Q-Lite/3Q-Pro/3Q-Max）。

**适用场景**：
- 日常文档质量检查（笔记/会议纪要）
- 重要文档深度检查（PRD/技能文档）
- 项目里程碑验收（发布/交付）
- 不确定用哪个 3Q 技能时

**质量评分**: 15/15（S 级技能）

**依赖关系**: 无（统一入口，内部调用框架）

**被依赖技能**: quality-os-trigger（自动触发）

## 🔧 智能识别引擎

### 识别流程

```
用户输入文档/内容
      ↓
┌─────────────────┐
│ 1. 文档类型检测  │
│ - 标题分析      │
│ - 关键词匹配    │
│ - 内容长度评估  │
└────────┬────────┘
         ↓
┌─────────────────┐
│ 2. 重要性评估    │
│ - 文档类型权重  │
│ - 内容深度评分  │
│ - 影响范围评估  │
└────────┬────────┘
         ↓
┌─────────────────┐
│ 3. 框架匹配      │
│ - 规则引擎匹配  │
│ - 推荐检查框架  │
│ - 用户确认/覆盖 │
└────────┬────────┘
         ↓
调用对应检查框架（Lite/Pro/Max）
```

### 重要性评分算法

```python
def calculate_importance(doc):
    score = 0
    
    # 文档类型权重（0-40 分）
    if doc.type in ["PRD", "技能文档", "发布计划"]:
        score += 40
    elif doc.type in ["方案", "总结", "复盘"]:
        score += 30
    elif doc.type in ["学习笔记", "计划"]:
        score += 20
    else:  # 笔记/纪要
        score += 10
    
    # 内容深度评分（0-30 分）
    if doc.word_count > 5000:
        score += 30
    elif doc.word_count > 2000:
        score += 20
    elif doc.word_count > 500:
        score += 10
    
    # 影响范围评分（0-30 分）
    if "发布" in doc.keywords or "验收" in doc.keywords:
        score += 30
    elif "方案" in doc.keywords or "决策" in doc.keywords:
        score += 20
    
    return score  # 0-100
```

### 框架匹配规则

```python
def match_framework(importance_score, doc_type):
    # 里程碑场景强制使用 Max
    if doc_type in ["里程碑", "发布", "验收"]:
        return "3Q-Max"
    
    # 根据重要性评分匹配
    if importance_score >= 70:
        return "3Q-Pro"
    elif importance_score >= 40:
        return "3Q-Lite"
    else:
        return "3Q-Lite"
```

## 📁 检查框架详情

### 3Q-Lite（三问框架）

**适用**：日常文档（笔记/纪要/日志）

**检查问题**：
1. **逻辑 Q** - 论证是否自洽？
   - 论据是否支持论点？
   - 有无逻辑漏洞？
   - 结论是否合理？

2. **用户 Q** - 用户价值是否清晰？
   - 解决了什么痛点？
   - 有无可感知的价值？
   - 表达是否清晰？

3. **竞争 Q** - 是否有差异化？
   - 与现有方案比有什么优势？
   - 独特价值是什么？

**评分标准**：
- **15 分** - S 级，可直接发布
- **14 分** - A 级，建议微调
- **13 分** - B 级，需要优化
- **<13 分** - 需要重大修改

**预计时间**：5-10 分钟

---

### 3Q-Pro（六问框架）

**适用**：重要文档（PRD/技能文档/方案）

**检查问题**：

**新增三问**（元认知层）：
1. **元认知 Q** - 任务理解是否正确？
   - 我对这个任务的理解有 95% 把握吗？
   - 哪些地方还需要澄清？
   - 我是否在逃避提问？

2. **标杆 Q** - 对标最佳实践？
   - 这个领域前 1% 的人会怎么做？
   - 他们的思维模式和我有什么不同？
   - 我可以在哪些方面向他们学习？

3. **挑战 Q** - 逆向思维检验？
   - 如果我的假设完全错误，会怎样？
   - 最强反对意见是什么？
   - 有哪些潜在风险未考虑？

**继承三问**（基础层）：
4. **逻辑 Q** - 论证是否自洽？
5. **用户 Q** - 用户价值是否清晰？
6. **竞争 Q** - 是否有差异化？

**评分标准**：
- **15 分** - S 级，可直接发布
- **14 分** - A 级，建议微调
- **13 分** - B 级，需要优化
- **<13 分** - 需要重大修改

**预计时间**：30-45 分钟

---

### 3Q-Max（三阶段框架）

**适用**：项目里程碑（发布/验收/交付）

**检查阶段**：

**阶段 1：事前 3Q**（15 分钟）
- 任务理解确认
- 质量标准定义
- 风险识别

**阶段 2：事中 13 检查点**（1-3 小时）
- 需求完整性检查
- 技术方案评审
- 测试覆盖验证
- 文档完整性检查
- ...（详见 quality-prevention-milestone）

**阶段 3：事后 3Q**（15 分钟）
- 交付物验收
- 经验总结
- 改进建议

**评分标准**：
- **通过** - 所有检查点合格
- **有条件通过** - 有 minor 问题，需修复
- **不通过** - 有 major 问题，需重新检查

**预计时间**：2-4 小时
