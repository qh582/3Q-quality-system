## 📋 优化内容

### 1. 新增任务分类决策树

**5 个判断问题，量化分类标准**：

```
问题 1: 任务是否需要创造性思考？
├─ 是 → 问题 2
└─ 否 → 问题 4（机械类）

问题 2: 任务是否涉及多模块整合？
├─ 是 → 整合类（×2.0，3Q-Max）
└─ 否 → 问题 3

问题 3: 任务产出是否需要差异化？
├─ 是 → 创作类（×1.5，3Q-Pro）
└─ 否 → 决策类（×1.0，3Q-Decision）

问题 4: 任务是否有明确标准流程？
├─ 是 → 机械类（×0.5，3Q-Lite）
└─ 否 → 重新评估（可能是决策类）

问题 5: 任务决策的影响范围？
├─ 影响整个项目 → 决策类（×1.0，3Q-Decision）
└─ 影响局部 → 重新评估
```

### 2. 新增边界条件与例外处理

#### 混合类型任务处理

**方法 1: 主导类型判断法**
- 评估各类型占比
- 选择占比>50% 的类型
- 示例：开发功能（60% 创作 +40% 机械）→ 创作类

**方法 2: 分段处理法**
- 不同阶段用不同 3Q 版本
- 示例：设计阶段（创作类）+ 实现阶段（机械类）

**方法 3: 就高原则**
- 选择质量要求更高的类型
- 示例：机械 + 整合 → 整合类（S 级要求）

#### 小团队适配（1-3 人）

- 权重系数调整：×0.8（所有类型）
- 3Q 检查简化：快速流程（10 分钟）
- 文档要求：简化模板

#### 紧急任务处理

- 分类：快速判断（2 分钟）
- 3Q 版本：3Q-Lite（快速流程）
- 质量要求：B 级（10-11 分）即可
- 事后补检：48 小时内补充完整检查
